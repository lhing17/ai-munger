# AI Munger — 查理芒格投资分析 Agent 设计文档

> 日期：2026-07-10 | 状态：设计阶段 | 版本：1.0

## 1. 项目目标

将查理·芒格的投资思想转化成一个可辅助投资决策的 AI Agent。用户通过多轮对话与 Agent 交互，Agent 驱动数据工具和分析技能，最终生成交互式 HTML 投资分析报告。

### 核心能力

1. **多轮对话驱动** — 混合模式：自由问答 + 引导式深度分析
2. **A 股数据获取** — 基于腾讯行情 + 东方财富 API，零外部依赖
3. **芒格思维模型分析** — 6 大分析维度覆盖芒格核心投资原则
4. **交互式 HTML 报告** — 含雷达图、趋势图、价格区间建议、可折叠面板

### 目标市场

- **主要**：A 股市场
- **对标**：支持与全球行业龙头对比分析

---

## 2. 架构总览

采用 **5 层分层架构**，每层职责清晰、可独立开发测试：

```
👤 用户 → 💬 多轮对话
    ⬇️
🎯 编排层 — munger-orchestrator
    ↙️        ↘️
📡 数据层     🧠 分析层
    ⬇️           ⬇️
🔧 工具层     📊 报告层
```

**5 层定义：**

| 层 | 职责 | 物理形式 |
|---|---|---|
| 工具层 | 原始数据获取（HTTP API） | Python CLI 脚本 |
| 数据层 | 封装工具调用，标准化输出 | Claude Code Skills |
| 分析层 | 芒格思维模型分析 | Claude Code Skills |
| 编排层 | 对话管理、流程调度、结果汇总 | Claude Code Skills |
| 报告层 | HTML 报告生成 | Claude Code Skill + HTML 模板 |

## 4. 分层详细设计

### 4.1 工具层

#### ashare_data.py（已有，需改进）

当前有 4 个命令：`quote`、`financials`、`valuation`、`search`。

**改进项：**

1. **增加 `--json` 参数** — Agent 可获取结构化 JSON 而不是格式化文本
2. **用 `urllib` 替代 `curl`** — 消除平台依赖（Windows 兼容），保持零外部依赖
3. **增加 `--period` 参数** — 支持季报/半年报，不局限于年报

#### global_data.py（第一批新增）

- 输入：全球龙头股票代码（如 AAPL, TSM）
- 数据源：Yahoo Finance
- 输出：营收/利润/ROE/毛利率/PE/市值等关键指标
- 命令：`python tools/global_data.py AAPL`

#### 工具层设计原则

- 零外部依赖（仅 Python stdlib）
- 错误输出到 stderr，stdout 为纯 JSON
- 所有命令支持 `--json` 参数
- 网络超时 15s，失败返回结构化错误

---

### 4.2 数据层 — 2 个 Skills

#### a-share-data.md

- **职责**：封装所有 A 股工具调用，输出统一 JSON 数据结构
- **调用**：`python tools/ashare_data.py quote/financials/valuation/search`
- **标准化**：无论调哪个命令，输出给分析层的都是结构化数据对象
- **容错**：某个数据源不可用时标记 `missing`，不阻断流程

#### financial-query.md

- **职责**：财务数据交叉验证与深度查询
- **调用**：工具脚本 + WebSearch（备选）
- **验证**：同一指标多渠道交叉比对
- **标记**：每条数据必须标注置信度（✅ 多源一致 / ⚠️ 单源 / ❓ 估计值）

---

### 4.3 分析层 — 6 个 Skills（分两批）

#### 标准 Skill 文档结构

每个分析 Skill 遵循统一模板：

```markdown
---
name: skill-name
description: 一句话描述
---

# 角色
你是查理芒格风格的 XX 分析师...

# 输入要求
需要以下数据（由数据层 Skill 提供）：
- 字段1
- 字段2

# 分析规则
量化阈值和判断标准...

# 输出格式
Markdown 表格 + 每维度判断理由 + 总评

# 行业例外
特定行业的阈值调整...
```

---

#### 第一批（跑通全流程）

**Skill 1: quality-screen — 量化去劣筛选**

| 维度 | 详情 |
|---|---|
| 芒格映射 | "买好公司的第一步是排除烂公司" |
| 量化指标 | ROE 连续 5 年 > 12%、有息负债率 < 50%、自由现金流/净利润 > 0.6、毛利率稳定性、应收账款/营收比、商誉占比 |
| 输出 | 7 指标得分卡 + PASS/FAIL/CAUTION 判定 |
| 例外 | 金融行业不适用负债率指标；初创企业允许 ROE 波动 |

**Skill 2: moat-analysis — 护城河深度分析**

| 维度 | 详情 |
|---|---|
| 芒格映射 | "伟大的公司都有宽阔的护城河" |
| 分析维度 | 品牌溢价（毛利率 vs 行业）、转换成本、网络效应、规模优势、特许经营权 |
| 输出 | 护城河评分卡（1-10 分/维度）+ 宽度/深度/趋势 |

**Skill 3: safety-margin — 安全边际评估**

| 维度 | 详情 |
|---|---|
| 芒格映射 | "以合理价格买伟大公司，比以便宜价格买普通公司好得多" |
| 分析维度 | PE/PB/PS 分位数（5年/10年）、DCF 粗略估算、清算价值底线、股息率支撑 |
| 输出 | 安全边际等级 + 合理买入区间估算 |

#### 第二批（丰富分析深度）

**Skill 4: management-check — 管理层审查**

- 资本配置能力（ROIC vs WACC、并购记录）
- 股权激励合理性（稀释速度）
- 分红与回购政策
- 历史言行一致性
- 关联交易/质押风险

**Skill 5: inversion-test — 逆向思维验证**

- 主动寻找 5-10 种"如何失败"的情景
- 行业颠覆者威胁、监管风险、技术替代风险
- 客户/供应商集中度风险、关键人物风险

**Skill 6: global-benchmark — 全球龙头对标**

- 对标全球同行业龙头的 ROE/毛利率/估值/研发投入/国际化程度
- 差距分析与追赶可能性评估

### 4.4 编排层 — munger-orchestrator

#### 两大交互模式

| 模式 | 触发条件 | 行为 |
|---|---|---|
| 💬 自由对话 | 日常提问（ROE、PE 等） | 按需调用工具，不启动完整流程 |
| 📊 深度分析 | 用户明确要求分析 / 触发词 | 完整 Pipeline → 生成 HTML 报告 |

深度分析模式的触发词："分析一下""深度研究""看看这家""帮我评估"

#### 5 阶段工作流

```
Phase 0: 意图识别
  → 判断自由问答 / 深度分析 / 行业筛选 / 个股分析

Phase 1: 数据收集（并行）
  → 调用 a-share-data Skill（内部并行执行 quote + financials + valuation 三个查询）

Phase 2: 质量筛选（串行 — 把关第一步）
  → quality-screen → PASS/FAIL/CAUTION
  → ⚠️ FAIL 时直接告知用户并终止流程

Phase 3: 并行分析
  → moat-analysis | safety-margin（并行执行）
  → 第二批加入 management-check | inversion-test

Phase 4: 全球对标（按需触发）
  → 用户要求时 / 行业对标有价值时 → global-benchmark

Phase 5: 报告生成
  → 汇总所有结论 + 数据 → 填充模板 → Write 到 reports/
```

#### 编排器 Skill 结构

1. 角色定义
2. 意图路由表（关键词 → 模式映射）
3. 工作流定义（Phase 0-5 完整步骤）
4. 异常处理（数据失败降级、质量不通过终止、模式切换）
5. 交互规范（Skill 调用格式、进度反馈节奏、报告生成回复模板）

#### 与现有投资 Skills 的关系

| 已有 Skill | 关系 | 说明 |
|---|---|---|
| financial-data | 复用 | 数据查询规范可直接使用 |
| quality-screen | 参考 | 通用版本，我们基于它增加芒格定制阈值 |
| investment-research | 独立 | 四大师框架，本系统专注芒格一人 |
| industry-funnel | 参考 | 未来可改造为芒格版行业筛选 |
| investment-checklist | 参考 | 为 management-check 提供对照 |

#### 核心交互组件

| 组件 | 说明 | 技术 |
|---|---|---|
| 雷达图 | 6-7 维度质量指标全貌 | Chart.js radar |
| 趋势折线图 | 营收/利润/ROE/毛利率/FCF 5 年走势 | Chart.js line |
| 可折叠面板 | 点击展开详细推理过程 | HTML details/summary + CSS |
| 综合评分条 | 加权评分 + 颜色编码区间 | CSS 渐变进度条 |

#### 综合评分与买入价格区间

**综合评分 = 6 维度加权汇总：**

| 维度 | 权重 | 满分 |
|---|---|---|
| 质量筛选 | 20% | 10 |
| 护城河宽度 | 25% | 10 |
| 安全边际 | 20% | 10 |
| 管理层质量 | 15% | 10 |
| 逆向风险 | 10% | 10 |
| 全球对标 | 10% | 10 |

**评级映射：**

| 分数 | 等级 | 说明 |
|---|---|---|
| 8-10 | 强烈推荐 | |
| 6-8 | 可以买入 | |
| 4-6 | 继续观察 | |
| < 4 | **回避** | 任何价格都不买 |

**买入价格区间（仅评级 ≥ 4 时展示）：**

| 类型 | 安全边际 | 估算方法 | 适用场景 |
|---|---|---|---|
| 🔥 激进型 | 15% | DCF 中枢 × 0.85 / PE 中枢 × 预测 EPS × 0.85 | 高成长确定性 + 强护城河 |
| ⚖️ 稳健型 | 30% | DCF 中枢 × 0.70 / PE 中枢 × 预测 EPS × 0.70 | 中等确定性 + 中等护城河 |
| 🛡️ 保守型 | 50% | DCF 中枢 × 0.50 / 清算价值底线 (Net-Net) | 要求极高的容错空间 |

**估值计算方法：**

1. **DCF 粗略估算** — 近 3 年 FCF 均值，3 种增长率（悲观/中性/乐观），折现率 10%
2. **PE 估值法** — 5 年 PE 中位数 × 趋势外推 EPS
3. **清算价值底线**（保守型参考）— (流动资产 - 总负债 + 固定资产 × 0.3) / 总股本
4. **交叉验证** — 2-3 种方法取交集/重叠区间

**不展示价格区间的情况**：评分 < 4、数据不足、盈利为负、FCF 持续为负

## 6. 核心设计原则

### 6.1 错误处理分层策略

| 层 | 策略 |
|---|---|
| 工具层 | 网络超时重试 1 次 → 返回错误 JSON；缺失字段标 null；`--json` 下错误结构化 |
| 数据层 | 工具错误 → 降级到 WebSearch；WebSearch 失败 → 标记 missing；标注置信度 |
| 分析层 | 缺关键数据 → "数据不足，无法判断"；评分默认为保守（缺数据 = 不给分） |

### 6.2 免责声明

所有报告必须内嵌：

> "本报告由 AI 基于公开数据自动生成，不构成投资建议。所有分析结论基于查理芒格的投资哲学框架，仅为信息参考。投资有风险，入市需谨慎。任何投资决策应由您独立判断并自行承担风险。"

- 每个数据指标标注来源
- 评分用区间而非精确值
- 买入价格用"估算"而非"建议买入价"
- 不预测股价走势

### 6.3 设计约定

- **每层可独立测试** — 不用启动全系统就能验证一个工具或一个分析 Skill
- **Skill = prompt 模板 + 调用规范** — 每个 Skill 是 Claude Code 的 skill 文件，不是独立服务
- **数据脚本零依赖** — 工具层只依赖 stdlib
- **渐进增强** — 先用最小集跑通全流程，再丰富分析维度和数据源

### 6.4 测试策略

1. **工具层**：每个命令独立执行验证 JSON 输出
2. **数据层**：手动触发 Skill，检查数据结构
3. **分析层**：用已知公司（如 茅台 600519）逐 Skill 测试
4. **端到端**：编排器 → 分析 → 报告 → 检查 HTML 完整性
5. **回归**：保存基准报告作为参考基线
