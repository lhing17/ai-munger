# AI Munger 三层门禁拒绝架构 — 设计规格

**日期**: 2026-07-10
**状态**: 设计完成，待 Review
**目标**: 解决"太难"堆缺失——系统总是分析，芒格大多数时候不分析

---

## 问题陈述

当前系统对所有标的一律启动完整的 5 阶段深度分析流程并产出 HTML 报告。这背离了芒格投资哲学的核心原则——"大部分公司的命运是进入'太难'堆，根本不值得分析"。系统被设计为"分析任何股票"，而芒格的天赋是"知道什么时候不分析"。

## 目标

- 60-80% 的分析请求在 1 分钟内被拒绝（产有价值的拒绝报告）
- 仅 20-40% 的标的进入完整深度分析流程
- 每一次拒绝都是一次成功的系统行为，不是错误
- 拒绝输出为投资者提供独立价值（信息卡 + 研究路线图）

## 架构：三层门禁

用户输入 "分析 X 公司"
→ Phase 0: 意图识别
→ Phase 1: 数据收集（5 路并行）
→ Gate 1: 能力圈审查 ← 新增
→ Gate 2: 质量筛选 ← 现有，轻微调整
→ Gate 3: 估值合理性 ← 新增
→ Phase 3-5: 完整分析（仅 ~20-40% 标的到达）

每个 Gate 独立判定，输出: PASS → 下一 Gate / 拒绝 → 终止流程

---

## Gate 1: 能力圈审查 (`circle-of-competence`)

**新增 Skill**: `skills/circle-of-competence.md`

**位置**: Phase 1（数据收集）之后，Phase 2（质量筛选）之前，记为 Phase 1.5

**职责**: 回答芒格第一问——"我懂这个生意吗？"

### 输入
- Phase 1 全部数据收集结果
- 行业分类（从 company description 推断或 WebSearch 确认）
- 用户可能提供的额外上下文

### 行业黑名单（第一层，快速过滤）

以下任一命中即标记 TOO-HARD：

| 类型 | 示例 | 原理 |
|------|------|------|
| 强政策博弈 | 军工、城投平台、K12教育 | 政策风险无法量化 |
| 技术迭代过快 | AI 芯片、基因编辑、量子计算 | 技术不确定性超过财务分析可覆盖范围 |
| 财务信息结构性缺位 | 壳公司、部分中小房企、上市 < 2年 | 数据不足以做严肃判断 |
| 商业模式不可验证 | 无主营收入、靠补贴生存 | 财务数据失去意义 |

### 信息质量评分（第二层，不在黑名单上的行业）

4 个维度，每维 0-2.5 分，满分 10：

| 维度 | 2.5 分 | 1.5 分 | 0.5 分 |
|------|--------|--------|--------|
| 财务数据完整性 | 5 年齐全，可交叉验证 | 3 年可用，部分缺口 | 严重不足 |
| 业务可理解性 | 商业模式简单清晰 | 需一定行业知识 | 极其复杂/不透明 |
| 信息真实性信心 | 审计无保留，国企/龙头 | 有小瑕疵 | 关联交易复杂，频繁换审计师 |
| 行业可预测性 | 需求稳定，格局清晰 | 有一定周期性 | 剧烈变化，难以预见 |

### 判定逻辑

| 条件 | 判定 |
|------|------|
| 总分 ≥ 7.0 且无黑名单命中 | **IN-CIRCLE** → Gate 2 |
| 总分 4.0-6.9 或黑名单命中 1 项 | **RESTRICTED** → 受限分析（仍执行 Gate 2 评估以丰富报告内容，但无论 Gate 2 结果如何都走拒绝路径） |
| 总分 < 4.0 或黑名单命中 ≥ 2 项 | **TOO-HARD** → 完整拒绝（跳过 Gate 2/3，直接产出拒绝报告） |
| 任一维度 0.5 分 | 触发"数据红线"，最多 RESTRICTED |

### 输出格式

**IN-CIRCLE**: 通过，传递可分析性评分至后续流程。

**TOO-HARD**: 完整拒绝报告，包含：
- 黑名单命中项 / 信息质量评分卡
- 已知信息卡（基础财务数据，不评分不推荐）
- 研究路线图：
  1. 这个生意到底靠什么赚钱？能一句话说清楚吗？
  2. 这个行业 10 年后还存在吗？竞争格局会变吗？
  3. 如果能拿到更多数据，什么数据最重要？
  4. 这家公司最可能被什么杀死？你能排除这个可能性吗？

**RESTRICTED**: 受限分析报告（同上 + 已知信息卡，不进入深度分析）

## Gate 2: 质量筛选 (`quality-screen`)

**现有 Skill**: `skills/quality-screen.md` — 基本保持不变

**调整项**:
- 从第一道（也是唯一一道）门禁变为第二道门禁
- 在 FAIL 时产出"质量拒绝报告"而非仅标记终止
- 与 Gate 1 的矩阵式判断：

| | IN-CIRCLE | RESTRICTED |
|---|---|---|
| **质量 PASS** | ✅ 完整分析 | ⚠️ 受限 + 能力圈警告 |
| **质量 FAIL** | 🚫 质量拒绝 | 🚫 太难 → 拒绝 + 路线图 |

- 具体规则和 7 项指标保持不变

## Gate 3: 估值合理性快速判断 (`valuation-gate`)

**新增 Skill**: `skills/valuation-gate.md`

**位置**: Phase 2（质量筛选 PASS/CAUTION）之后，Phase 3（深度分析）之前，记为 Phase 2.5

**职责**: 回答——"价格基本合理吗？还是离谱到不值得继续分析？"

**关键原则**: Gate 3 不替代 Phase 3 的 safety-margin 深度估值。它只是排除"明显不值得看"的标的。

### 输入
- 当前 PE(TTM)、PB
- 近 5 年 PE 中位数
- 行业平均 PE（如有）
- quality-screen 的 ROE/毛利率数据

### 判定标准

| 条件组合 | 判定 |
|----------|------|
| PE 在 5 年中位数 ±30% 内 | **REASONABLE** |
| PE 高于中位数 30%-80%，或 PE > 行业均值 2× | **EXPENSIVE** |
| PE 高于中位数 > 80%，或 PE > 50× 且 ROE < 15%，或 PB > 10× | **ABSURD** |
| 净利润为负且无扭亏预期 | **ABSURD** |

- **REASONABLE** → 通过，进入 Phase 3
- **EXPENSIVE** → 通过但标注，报告中提示"当前价格偏高"
- **ABSURD** → 终止流程，产出估值拒绝报告

### ABSURD 拒绝输出

```
💰 当前价格不在合理范围内

当前 PE: 85×(TTM) vs 5年中位数: 25×
ROE: 8%

芒格不会在任何接近当前价格的价位考虑这家公司。
这不是说公司不好——市场已经定价得太乐观了。
如果它跌到 YY 元（合理PE × 近1年EPS），我们再来分析。

眼下最好的行动：放入观察清单，等待价格回归。
```

## 统一拒绝输出系统

**新增 Skill**: `skills/rejection-report.md`

**核心原则**: 拒绝不是错误。是系统做了芒格会做的事。

### 统一拒绝报告结构

```
🚫 芒格判定：不投

拒绝来源: Gate 1 / 2 / 3
拒绝等级: TOO-HARD / NOT-QUALITY / TOO-EXPENSIVE

▼ 一句话结论（芒格口吻）

▼ 命中 Gate 的详细判定依据
  ├── Gate 1: 可分析性评分卡
  ├── Gate 2: 质量筛选得分卡
  └── Gate 3: 估值对比卡

▼ 已知信息摘要（不评分、不推荐）
  基础财务 + 行业 + 当前估值

▼ 研究路线图（如果 TOO-HARD）
  需要回答的核心问题清单

▼ 重新触发条件（如果 TOO-EXPENSIVE）
  什么价格可以再来分析

▼ 芒格语录（相关引用一句）

─────────────────────────
⚠️ 以上分析不构成投资建议
```

### 多 Gate 命中优先级

Gate 1 > Gate 2 > Gate 3

"不懂"比"不好"重要，"不好"比"太贵"重要。

### 拒绝 vs 完整分析分界线

| 路径 | 产出 | 预计耗时 |
|------|------|---------|
| Gate 1 TOO-HARD | 完整拒绝 + 研究路线图 | ~30s |
| Gate 1 RESTRICTED | 受限报告 = 拒绝 + 已知信息卡 | ~45s |
| Gate 2 FAIL | 质量拒绝报告 | ~45s |
| Gate 3 ABSURD | 估值拒绝 + 观察清单建议 | ~45s |
| 全部通过 | 完整 HTML 报告（现有流程） | ~3-5min |

**目标**: 60-80% 标的在 1 分钟内完成拒绝。

## 非功能约束

1. **免责声明**: 所有拒绝报告同样包含免责声明
2. **拒绝不是失败**: 拒绝是成功的系统行为，用肯定的语气（"芒格不会投这个"而非"系统无法分析这个"）
3. **数据保守处理**: 缺数据时偏向拒绝（宁可错杀，不滥分析）
4. **不预测股价**: Gate 3 不给目标价，只给"值得再来分析的价格区间"
5. **芒格风格**: 拒绝报告同样要求简洁直接、逆向思维、承认无知

---

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 用户认为系统"什么都不肯分析" | 拒绝输出提供独立价值（研究路线图、信息卡）；RESTRICTED 路径提供受限分析 |
| 行业黑名单过于粗糙，误杀好公司 | 黑名单命中仅 1 项且信息质量评分高 → 走 RESTRICTED 而非 TOO-HARD |
| Gate 3 过于敏感，好公司在调整期被拒 | 阈值设置保守：PE 超中位数 80% 才 ABSURD；EXPENSIVE 通行 |
| orchestrator 逻辑复杂度过高 | 每个 Gate 独立 Skill，orchestrator 只做编排和分支；Gate 间通过统一数据格式传递 |
